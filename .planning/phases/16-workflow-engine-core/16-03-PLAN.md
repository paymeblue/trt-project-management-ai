## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-05 | Elevation of Privilege | a user completing/skipping a step that is not theirs | mitigate | Every action gates with verifySession + canRoleActOnStep(step.role, role) before any write |
| T-16-06 | Tampering | client bypassing required-step gate by passing skip=true | mitigate | Skip enforcement lives server-side in completeGraphStep (throws on required); the client flag cannot override it |
| T-16-07 | Spoofing | self-approval on a two-party approval step | mitigate | receiveApproval rejects when receiver == sender (two-party rule) |
| T-16-08 | Tampering | assigning a user of the wrong role | mitigate | assignUser verifies the assignee's role == step.targetRole |
</threat_model>
