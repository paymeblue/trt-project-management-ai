**Areas discussed:** Trigger UX, Security trade-off, Sign-out scope

---

## Trigger UX

| Option | Description | Selected |
|--------|-------------|----------|
| Explicit "sign in as different user" action | Normal tab-opening still inherits the current session; a specific action starts a fresh, independent per-tab session | ✓ |
| Every new tab starts logged out by default | No tab ever inherits another tab's session automatically; every new tab requires fresh sign-in | |

**User's choice:** Explicit "sign in as different user" action (Recommended option).
**Notes:** Preserves today's convenience for the common single-user case; only deliberately switching identity in a tab triggers the new mechanism.

---

## Security trade-off

| Option | Description | Selected |
|--------|-------------|----------|
| Accept it, short-lived tokens | sessionStorage + Authorization header, mitigate larger XSS blast radius with short token lifetimes + refresh | ✓ |
| Keep httpOnly cookies, server-side tab-session table | Cookie holds only an opaque per-tab session ID; server maps ID to user in a sessions table | |

**User's choice:** Accept it, short-lived tokens (Recommended option).
**Notes:** Judged acceptable given TRT PM is an internal tool with a small, known user base.

---

## Sign-out scope

| Option | Description | Selected |
|--------|-------------|----------|
| Sign out only that tab | Each tab manages its own session lifecycle independently | ✓ |
| Sign out everywhere for that user | Signing out in any tab invalidates that user's session across all tabs | |

**User's choice:** Sign out only that tab (Recommended option).
**Notes:** Matches full per-tab independence; avoids needing server-side token revocation/blocklist for this phase.
