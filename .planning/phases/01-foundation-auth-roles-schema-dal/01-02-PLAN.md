<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> Server Action | untrusted FormData (incl. role field) crosses here |
| browser -> protected route | session cookie presented; may be absent/forged |
| Server Action -> DB | every mutation must re-authorize regardless of UI |
| CLI -> Neon Auth admin API | seed script holds elevated privileges; never web-exposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Elevation of Privilege | signupAction role field | mitigate | `z.enum(['factory_pm','site_pm'])` server-side whitelist; super_admin absent from actions/auth.ts (grep `super_admin` count == 0); reject before any auth.signUp call |
| T-01-06 | Elevation of Privilege | cross-role route access | mitigate | Route-group layouts call `requireRole()` via DAL -> forbidden(); proxy is presence-only and never the sole defense |
| T-01-07 | Tampering | Server Action invoked directly (bypassing proxy/UI) | mitigate | DAL verifySession + requireOwnerOrAdmin is the first call in every mutation; proxy excluded from the authorization decision |
| T-01-08 | Spoofing | session cookie tampering | mitigate | Neon Auth signs the cookie with NEON_AUTH_COOKIE_SECRET (HMAC); proxy + DAL both call auth.getSession() which verifies the signature |
| T-01-09 | Spoofing | role claim lag after signup | mitigate | await db.insert(users) BEFORE auth.signIn (pitfall 5); DAL reads role from Postgres, never from a JWT claim |
| T-01-10 | Elevation of Privilege | super_admin self-provisioning via web | mitigate | super_admin created only by db/seed-admin.ts (CLI, no 'use server', not a route); auth.admin.createUser never called from request-handling code |
| T-01-11 | Information Disclosure | NEON_AUTH_COOKIE_SECRET exposure | mitigate | `import 'server-only'` on lib/auth/server.ts; no NEXT_PUBLIC_ prefix; secret read only via process.env in server code |
</threat_model>
