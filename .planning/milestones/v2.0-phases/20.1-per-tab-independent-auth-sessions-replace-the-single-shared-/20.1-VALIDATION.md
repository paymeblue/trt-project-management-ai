## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Vitest (per `package.json`, config at `vitest.config.ts`) |
| **Config file** | `vitest.config.ts` — `environment: 'node'`, includes `tests/**/*.test.ts`, `lib/**/*.test.ts`, `app/**/*.test.ts` |
| **Quick run command** | `npx vitest run <touched test file>` |
| **Full suite command** | `npm test` (= `vitest run`) |
| **Estimated runtime** | ~30-60 seconds |

Existing convention (confirmed via `tests/actions/auth.test.ts`): mock `server-only`, `next/navigation` (`redirect` throws `Error('REDIRECT')`), `next/headers` (`cookies()`), and `@/auth` (`signIn`/`signOut`) via `vi.mock`. New tests follow this pattern, additionally mocking `next-auth/jwt`'s `encode`/`decode` and `next/headers`'s `headers()`.

---

## Sampling Rate

- **After every task commit:** Run `npx vitest run <touched test file>`
- **After every plan wave:** Run `npm test`
- **Before `/gsd:verify-work`:** Full suite must be green, PLUS the manual A1 smoke test (real browser, two tabs, two different users) — this phase's core claim ("independent per-tab sessions actually work") cannot be fully proven by unit tests alone; the original bug was itself only caught by live/manual reproduction.
- **Max feedback latency:** ~60 seconds
