**Extracted shared bcrypt credential verification, added per-tab JWE mint/verify primitives on next-auth/jwt (no new crypto dependency), and gave lib/dal.ts's verifySession() a header-first/cookie-fallback precedence that fails closed on invalid tokens — all 57 existing DAL call sites unchanged.**

## Accomplishments
- `verifyCredentials(email, password)` is now the single source of truth for bcrypt credential checks; `auth.ts`'s `Credentials.authorize()` delegates to it with zero duplicated compare logic
- `lib/tab-session.ts` mints and verifies per-tab access/refresh JWE tokens using Auth.js's own `next-auth/jwt` `encode`/`decode`, isolated by a distinct salt, with `typ`-based cross-use rejection
- `lib/dal.ts`'s `verifySession()` now resolves a valid `Authorization: Bearer` header ahead of the shared session cookie, fails closed on an invalid/expired/wrong-`typ` header, and leaves the cookie-only default path (all 57 call sites) completely untouched
- `auth.config.ts`'s `authorized()` proxy gate recognizes header presence (no decode) for the optimistic pass-through

**1. [Rule 3 - Blocking] Removed a type-widening cast in auth.config.ts's jwt() callback**
- **Found during:** Task 2 (before touching auth.config.ts's `authorized()` in Task 3), while running the plan's mandated `npx tsc --noEmit` verification
- **Issue:** `token.role = (user as { role?: string }).role` cast `user` down to an ad hoc `{ role?: string }` shape, discarding the project's `types/next-auth.d.ts` module augmentation (`User.role: Role`, non-optional) and producing `Type 'string | undefined' is not assignable to type 'UserRole'` / similar for `token.id`. - **Fix:** Removed the unnecessary narrowing cast; `user.id`/`user.role` now flow through as their already-augmented `string`/`Role` types (`token.id = user.id as string`, `token.role = user.role as UserRole`)
- **Files modified:** `auth.config.ts`
- **Verification:** `npx tsc --noEmit` clean before and after all subsequent tasks
- **Committed in:** `b8b0e08` (standalone `fix` commit, separate from the Task 2/3 `feat` commits)
