**Analog:** itself, `/Users/uzochukwuamara/Code/PayMeBlue/trt-project-manager/trt-pm/lib/dal.ts` (full file, 35 lines — read in one pass)

**Current full pattern (imports + core + cache wrapping)** (lines 1-14):
```typescript

import { cache } from "react"
import { redirect, forbidden } from "next/navigation"
import { auth } from "@/auth"
import { isAdminRole, type UserRole } from "@/lib/workflow"

export type Role = UserRole
export { isAdminRole }

export const verifySession = cache(async (): Promise<{ userId: string; role: Role }> => {
  const session = await auth()
  if (!session?.user?.id) redirect("/sign-in")
  return { userId: session.user.id, role: session.user.role as Role }
})
```

**Role-gate pattern to preserve unchanged** (lines 16-34):
```typescript
export async function requireRole(role: Role) {
  const s = await verifySession()
  if (s.role !== role) forbidden()
  return s
}

export async function requireAdmin() {
  const s = await verifySession()
  if (!isAdminRole(s.role)) forbidden()
  return s
}

export async function requireOwnerOrAdmin(ownerId: string) {
  const s = await verifySession()
  if (s.role === "super_admin") return s
  if (s.userId !== ownerId) forbidden()
  return s
}
```

**What changes (per RESEARCH.md Pattern 2 / D-06):** `verifySession()`'s body gets a header-first check inserted before the existing `auth()` call — add `import { headers } from "next/headers"` and `import { verifyTabToken } from "@/lib/tab-session"`, check `(await headers()).get("authorization")` first, `decode()`/`verifyTabToken()` it, and only fall through to the existing `auth()` call if no header is present. `requireRole`/`requireAdmin`/`requireOwnerOrAdmin` call `verifySession()` internally and need **zero changes** — this is the entire point of the DAL choke-point pattern already established here.

**Do not** call Auth.js's `getToken()` — it checks cookie before header (opposite precedence needed). Build the check manually as shown in RESEARCH.md Pattern 2.

**Pattern to follow (adapted, not copied verbatim — this repo's convention for a "server-only crypto/token service" module):**

```typescript
import "server-only"
import { encode, decode } from "next-auth/jwt"

const TAB_SALT = "trt-pm.tab-session"

export async function mintTabAccessToken(userId: string, role: string) {
  return encode({
    secret: process.env.AUTH_SECRET!,
    salt: TAB_SALT,
    maxAge: 20 * 60,
    token: { sub: userId, role, typ: "access" },
  })
}

export async function verifyTabToken(token: string) {
  return decode({ secret: process.env.AUTH_SECRET!, salt: TAB_SALT, token })
}
```

import bcrypt from "bcryptjs"

async function authorize(creds) {

}

### `app/api/auth/tab-refresh/route.ts` (route/Route Handler, request-response) — NEW

**Analog:** `/Users/uzochukwuamara/Code/PayMeBlue/trt-project-manager/trt-pm/app/api/my-work/route.ts` (full file, 15 lines)

**Full pattern to copy (imports, `dynamic` export, DAL call, `NextResponse.json` shape):**
```typescript
import { NextResponse } from 'next/server'
import { verifySession } from '@/lib/dal'
import { getMyWork } from '@/lib/my-work'
import type { UserRole } from '@/lib/workflow'

export const dynamic = 'force-dynamic'

export async function GET() {
  const { role, userId } = await verifySession()
  const work = await getMyWork(role as UserRole, userId)
  return NextResponse.json(work, { headers: { 'Cache-Control': 'no-store' } })
}
```

**Adapted for tab-refresh (POST, not GET; reads a refresh token, not the session):**
```typescript
import { NextResponse } from 'next/server'
import { verifyTabToken, mintTabAccessToken } from '@/lib/tab-session'

export const dynamic = 'force-dynamic'

export async function POST(req: Request) {
  const { refreshToken } = await req.json()
  const payload = await verifyTabToken(refreshToken)
  if (!payload?.sub || payload.typ !== 'refresh') {
    return NextResponse.json({ error: 'Invalid refresh token' }, { status: 401 })
  }
  const accessToken = await mintTabAccessToken(payload.sub as string, payload.role as string)
  return NextResponse.json({ accessToken, expiresAt: Date.now() + 20 * 60 * 1000 })
}
```
Note: this Route Handler must NOT call `verifySession()` (the analog's pattern) — it authenticates via the refresh token in the request body, not via the shared-cookie/header DAL path, since its entire job is to mint a *new* header-path credential.

### `app/_components/tab-session-provider.tsx` (provider/client component, event-driven) — NEW

**Analog:** `/Users/uzochukwuamara/Code/PayMeBlue/trt-project-manager/trt-pm/app/_components/my-work-provider.tsx` (full file, 67 lines)

**`'use client'` + polling-provider shape to copy** (lines 1-19, 44-57):
```typescript
'use client'

import { createContext, useCallback, useContext, useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

const Ctx = createContext<MyWorkContext>({ /* defaults */ })
export function useMyWork() { return useContext(Ctx) }

const POLL_MS = 4000

export default function MyWorkProvider({ initial, children }) {
  const [data, setData] = useState(initial)
  const pathname = usePathname()

  const refresh = useCallback(async () => {
    try {
      const res = await fetch('/api/my-work', { cache: 'no-store' })
      if (!res.ok) return
      const json = await res.json()
      setData(json)
    } catch {
      // transient network error — keep last known state
    }
  }, [])

  useEffect(() => {
    const id = setInterval(refresh, POLL_MS)
    const onVisible = () => { if (document.visibilityState === 'visible') refresh() }
    document.addEventListener('visibilitychange', onVisible)
    window.addEventListener('focus', refresh)
    return () => {
      clearInterval(id)
      document.removeEventListener('visibilitychange', onVisible)
      window.removeEventListener('focus', refresh)
    }
  }, [refresh])

  return <Ctx.Provider value={{ ...data, refresh }}>{children}</Ctx.Provider>
}
```

**Mount point to replicate:** `MyWorkProvider` is mounted in `app/(app)/layout.tsx` (line 16 import, wraps `children` further down that file) — `TabSessionProvider` should mount at the same layer (root `app/layout.tsx` per RESEARCH.md's structure recommendation, since it must also activate for the sign-in page itself, which sits outside `app/(app)/`). Verify exact mount layer during planning since `app/(app)/layout.tsx` gates on `auth()` + redirects unauthenticated users before rendering, which would prevent the provider from ever mounting on `/sign-in`.

**Divergence (per RESEARCH.md Pattern 4):** this provider does not poll — it installs a `window.fetch` override in a `useEffect`, gated on `sessionStorage.getItem('tabAccessToken')` being present (default single-session tabs: no-op, untouched `fetch`). See RESEARCH.md Pattern 4's full code block. The `useEffect` cleanup (`return () => { window.fetch = originalFetch }`) mirrors the analog's cleanup-function convention (lines 52-56 above) even though the mechanism (override, not interval/listener) differs.

## Shared Patterns

### Server-only module marker
**Source:** `lib/dal.ts` line 1, `lib/auth/email-flows.ts` line 1
**Apply to:** `lib/tab-session.ts`, `lib/auth/verify-credentials.ts` — every new module that touches `AUTH_SECRET`, bcrypt, or DB user lookups must start with `import "server-only"` to prevent accidental client bundling, matching 100% of existing precedent in `lib/`.

### `'use server'` + zod-validated Server Action result-state shape
**Source:** `actions/auth.ts` lines 1, 21-23, 44-47
**Apply to:** `actions/tab-auth.ts` — copy the `{ message?: string }`-shaped state type + `z.object({ email: z.email({error}), password: z.string().min(1, {error}) })` validation exactly; only the return payload on success differs (tokens instead of nothing/redirect).

### DAL as sole authoritative choke point
**Source:** `lib/dal.ts` (all of it) + CONTEXT.md's "Established Patterns" section
**Apply to:** `app/api/auth/tab-refresh/route.ts` is the one deliberate exception (it must NOT call `verifySession()`, see note above) — every other new/modified Route Handler, Server Component, and Server Action in this phase must go through `verifySession()`/`requireRole()`/`requireAdmin()`/`requireOwnerOrAdmin()` unchanged, never `auth()` or JWT `decode()` directly.

### Client component + `useEffect` cleanup for a global side effect
**Source:** `app/_components/my-work-provider.tsx` lines 45-57
**Apply to:** `app/_components/tab-session-provider.tsx` — same `useEffect(() => { ...install...; return () => {...uninstall...} }, [deps])` shape, even though this provider overrides `window.fetch` instead of setting up interval/event listeners.

### Route Handler minimal shape
**Source:** `app/api/my-work/route.ts` (full file)
**Apply to:** `app/api/auth/tab-refresh/route.ts` — `export const dynamic = 'force-dynamic'`, `NextResponse.json(...)` return, no try/catch wrapper in the analog (errors bubble to Next's default error handling) — match this minimalism rather than adding a bespoke error-handling layer.
