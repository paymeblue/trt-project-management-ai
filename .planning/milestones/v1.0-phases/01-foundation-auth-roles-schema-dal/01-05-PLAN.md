<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> signUpAction | untrusted FormData (incl. role field) crosses here |
| browser -> protected route | NextAuth JWT cookie presented to a role-gated layout; may be absent/forged |
| Server Action -> DB | every mutation must re-authorize regardless of UI |
| CLI -> seed insert | super_admin provisioning holds elevated privilege; never web-exposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Elevation of Privilege | signUpAction role field | mitigate | `z.enum(['factory_pm','site_pm'])` server-side whitelist; super_admin absent from actions/auth.ts (grep count == 0); reject before any hash/insert/email |
| T-01-06 | Elevation of Privilege | cross-role route access | mitigate | Role-group layouts call `requireRole()` via the DAL -> forbidden(); proxy is optimistic and never the sole defense |
| T-01-07 | Tampering | Server Action invoked directly (bypassing proxy/UI) | mitigate | DAL verifySession + requireRole/requireOwnerOrAdmin is the authoritative gate in gated layouts/actions; proxy excluded from the decision |
| T-01-08 | Spoofing | role claim lag after signup | mitigate | await db.insert(users) BEFORE signIn; DAL reads role from the verified JWT (set from the DB user at authorize time), not from client input |
| T-01-09 | Elevation of Privilege | super_admin self-provisioning via web | mitigate | super_admin written ONLY by db/seed-admin.ts (CLI, no 'use server', not a route); NextAuth exposes no admin HTTP API |
| T-01-10 | Information Disclosure | password / hash exposure in signup path | mitigate | bcrypt.hash(password,10) before insert; only the hash is stored; raw password never logged; sign-in error is generic |
| T-01-11 | Information Disclosure | ADMIN_PASSWORD / AUTH_SECRET exposure | mitigate | seed reads ADMIN_PASSWORD from process.env only (CLI); no NEXT_PUBLIC_; secrets never client-bundled |
</threat_model>
