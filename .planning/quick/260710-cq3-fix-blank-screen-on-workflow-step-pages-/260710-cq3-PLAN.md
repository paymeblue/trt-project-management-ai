<objective>
Kill the blank-screen failure mode on `/workflow/step` and `/admin/payment-confirmation`.

Root cause (diagnosed from the code, not guessed):
1. `/admin/payment-confirmation/page.tsx` calls `requireAdmin()`, which calls
   `forbidden()` for any non-admin (e.g. a Customer Care user). `authInterrupts`
   is enabled in `next.config.ts`, but there is NO `forbidden.tsx` boundary
   anywhere in `app/` — so `forbidden()` yields a blank screen with no redirect.
2. Both step pages return inline "message" states (missing params, unknown step,
   "not your turn", wrong position, project-not-found, not-awaiting-payment) that
   only render a Dashboard *link*, not a guaranteed redirect.
3. There is NO `error.tsx` under `(app)`, so any thrown error in these
   `force-dynamic` server pages also blanks out.

Fix: (a) replace every "cannot render meaningful content" branch with a
server-side `redirect()` to the user's role dashboard, (b) add contextual
`loading.tsx` skeletons for both routes, and (c) add an `(app)/error.tsx`
safety net so unforeseen throws never blank the screen.

Purpose: A user who lands on a step page they cannot use always ends up
somewhere real (their dashboard) with a loading indicator in between — never a
blank void.
Output: 2 edited pages + 2 loading files + 1 error boundary.
</objective>
