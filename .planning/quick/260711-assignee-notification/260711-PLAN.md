<objective>
Close the ROLE-02 gap surfaced by Phase 19's verification harness: when a user is
assigned to a workflow step via an `assignment`-kind step, the assignment is recorded
but the assignee is never notified. `scripts/verify-role-assignment.ts` proved this by
asserting a `notifications` row should exist for the assignee after assignment and
finding none.

Fix: add a single-recipient `notifyUser` to `lib/notifications.ts` (mirroring the
existing `notifyAllSuperAdmins` fan-out shape), fire it from `assignUser` in
`lib/workflow-graph.ts` right after the assignment is durably recorded, and surface it
to the assignee by mounting the header bell for every authenticated role (today it is
super_admin-only, so an assignee could never see the row — an invisible row does not
"notify that user," which ROLE-02 requires).

Purpose: honestly complete ROLE-02's "notifies that user they've been assigned" half
using the existing in-app notification system — not a new one.

Output: `notifyUser` function, wired assignment notification, assignee-visible bell,
a tightened verification assertion that now PASSes, and ROLE-02 flipped to Complete.
</objective>
