<objective>
Fix the dead-end after assigning an officer/PM on a workflow assignment step. Give a clear confirmation ("✓ {name} assigned to \"{step}\" on {project}") and redirect to the actor's role dashboard, so the assigner knows it worked and lands somewhere sensible.

Purpose: Replace silent/ambiguous feedback with an explicit success message + redirect.
Output: Enriched `AssignmentStep` component + the step page wiring the context props into it.

Scope guard: UX only. Do NOT add unassign/reassign, and do NOT change the workflow engine (`lib/workflow-graph.ts`) or the server actions' behavior. The assignment step must remain pending/completable from the dashboard after redirect (project currentStep is unchanged by Assign), so a future reassign task is not blocked.
</objective>
