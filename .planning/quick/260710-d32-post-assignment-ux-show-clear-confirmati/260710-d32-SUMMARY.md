# Phase quick-260710-d32: Post-assignment UX — show clear confirmation Summary

**AssignmentStep now shows a named "who was assigned to what step (and project)" confirmation and auto-redirects to the actor's role dashboard ~1.4s after a successful Assign or Complete, replacing the bare "Assigned." dead-end.**

## Accomplishments
- `AssignmentStep` accepts optional `stepLabel`, `projectName`, and `redirectTo` props.
- On successful Assign, the message reads `✓ {assigneeName} assigned to "{stepLabel}" on {projectName}. Redirecting…` (assignee name is looked up from `candidates` by the currently `selected` id; project clause omitted when `projectName` is absent; "Redirecting…" omitted when `redirectTo` is absent).
- On successful Complete, the message reads `✓ Step completed. Redirecting…` (or without the redirect clause if no `redirectTo`).
- On failure of either action, the existing error-message behavior is preserved (`res.message ?? 'Could not assign.'` / `'Could not complete step.'`) with no redirect.
- After a successful action with `redirectTo` set, `router.push(redirectTo)` fires ~1400ms later so the confirmation is readable first; the timer is stored in a ref and cleared on unmount via a `useEffect` cleanup to avoid navigating/setting state after the component is gone.
- Success messages render in green (`text-green-700`); failure messages reuse the existing `text-error` class.
- `app/(app)/workflow/step/page.tsx` now fetches the project's `name` once (via `db.select({ name: projects.name }).from(projects).where(eq(projects.id, projectId!)).limit(1)`, after the `!projectId || !stepKey` redirect guard so `projectId!` is safe) and passes `stepLabel={step!.label}`, `projectName={projectName}`, and `redirectTo={dashboard}` into `<AssignmentStep>`.
- No unassign/reassign UI was added; `assignUserAction`/`completeStepAction` and the workflow engine (`lib/workflow-graph.ts`) are untouched — the assigned step remains pending/completable from the dashboard after redirect.
