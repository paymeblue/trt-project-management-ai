  <name>Task 1: Pure countdown formatter + unit tests + client component</name>
  <files>lib/countdown.ts, tests/lib/countdown.test.ts, app/_components/deadline-countdown.tsx</files>
  <behavior>
    Pure formatCountdown(deadline: string | null, now: number) returns:
    { hasDeadline: boolean; overdue: boolean; text: string; tier: 'normal' | 'warn' | 'urgent' }

    - deadline === null → { hasDeadline: false, overdue: false, text: '', tier: 'normal' }
    - Let ms = new Date(deadline).getTime() - now; overdue = ms < 0.
    - Break Math.abs(ms) into d / h / m / s; zero-pad h, m, s to 2 digits.
    - Text: when d > 0 → `${d}d ${hh}:${mm}:${ss}` (e.g. "3d 04:12:05");
      when d === 0 → `${hh}:${mm}:${ss}` (e.g. "04:12:05");
      when overdue → prefix "Overdue " (e.g. "Overdue 1d 02:31:07").
    - Tier (by remaining time, NOT absolute): overdue → 'urgent';
      else ms < 6h → 'urgent'; else ms < 24h → 'warn'; else → 'normal'.

    Test cases (tests/lib/countdown.test.ts, using a fixed `now` and computed deadlines):
    - null deadline → hasDeadline false, text '', tier normal
    - ~3d 4h 12m 5s ahead → text "3d 04:12:05", overdue false, tier 'normal'
    - ~30h ahead → tier 'normal'
    - ~12h ahead → tier 'warn', text has no "d " prefix, format "HH:MM:SS"
    - ~4h 12m 5s ahead → text "04:12:05", tier 'urgent'
    - ~1d 2h 31m 7s past due → text "Overdue 1d 02:31:07", overdue true, tier 'urgent'
    - single-digit parts pad: ~1h 2m 3s ahead → text "01:02:03"
  </behavior>

    Zero-pad with String(n).padStart(2, '0'). Use constant ms divisors
    (86_400_000 / 3_600_000 / 60_000 / 1000), mirroring project-steps-board.tsx's
    Countdown math.
