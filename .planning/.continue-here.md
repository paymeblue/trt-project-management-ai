## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Blind `drizzle-kit push` | Applies the ENTIRE schema.ts↔DB diff, not just the change you intended — can silently destroy unrelated data | blocking | Use targeted raw SQL (`ALTER TABLE ... ALTER COLUMN`, explicit constraint drop/recreate) verified with a read-back query instead |
| Deleting users with FK-referenced history | Most `created_by`/`completed_by`/`assigned_user_id` columns are still NOT NULL with no cascade — deleting a user who's ever created/completed/been-assigned anything fails loudly (safe) rather than silently corrupting data | advisory | This is actually the correct safety net — don't "fix" it by adding cascading deletes without being asked; only relax specific columns (like `checklists.createdBy` this session) when explicitly requested and the semantics support it |
